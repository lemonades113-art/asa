## 🟢 基础功能验证集

### P1: 元数据标签验证
**Prompt:**
```
查询贵州茅台(600519)2025年11月20日至30日的日线数据，
按时间升序排列，显示开盘价、最高价、最低价、收盘价、成交量。
```

**预期验证点：**
- ✅ 返回结果包含 `[DATE]: yyyy-mm-dd` 标签
- ✅ 返回结果包含 `[SOURCE]: Tushare` 标签
- ✅ 返回结果包含 `[META]: 股票=600519, 周期=D, 记录数=10` 等信息
- ✅ 返回结果包含 `[DATA]:` 标签且后续是表格数据

**验证命令：**

**难度：** ⭐⭐ （最容易，直接查看返回结果）

### P4: 网络错误 vs 代码错误的独立计数
**Prompt:**
```
在高峰期（如14:55）查询当日股票实时数据。
注意：此时Tushare可能有速率限制。
```

**预期验证点：**
- ✅ 如果返回429/503错误，ErrorHandler分类为`network_error`
- ✅ network_retry_count独立计数（≠ retry_count）
- ✅ 网络错误使用指数退避策略（2^n秒延迟）
- ✅ 即使网络重试失败，代码重试计数仍不增加

**验证命令：**
```bash
# 查看日志中
[ErrorHandler] 错误分类：network_error
[ErrorHandler] 指数退避延迟：4秒
[State] retry_count: 0, network_retry_count: 1
```

**难度：** ⭐⭐⭐⭐ （需要造成网络错误，难度最高）

### P12: realtime vs daily缓存TTL差异
**Prompt:**

## ⚠️ 常见验证陷阱

1. **缓存干扰**：如果Prompt P12显示缓存命中，可能是之前的测试数据
   - 解决：清空缓存，用新的股票代码或日期范围

2. **模型行为不确定性**：LLM的输出可能有变异
   - 解决：运行多次，看主要流程是否一致

3. **API限制**：Tushare可能有速率限制或数据权限限制
   - 解决：检查API-KEY有效性，降低查询频率

4. **时间依赖**：某些测试对时间敏感（如"当日数据"）
   - 解决：使用历史日期，而不是实时日期

5. **跨会话状态**：如果ProfileUpdater没有数据库持久化，跨会话学习不工作
   - 解决：确认系统设计（是否需要添加数据库）
