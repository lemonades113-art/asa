## 项目概述

**项目名称**：A股数据分析助手 (Multi-Agent Trading Analysis System v2.1)

**开发目的**：构建一个多智能体协作系统，用于A股市场的数据分析、财务评估和投资建议生成

**核心模式**：LangGraph Supervisor模式 + 自我修正机制 + 链式思考可视化

**技术栈**：
- 后端框架：LangGraph (多智能体编排)
- LLM模型：Qwen-Plus (强逻辑) + Qwen-Turbo (轻量级) - 阿里云通义千问
- Web框架：Gradio 5.18.0
- 数据接口：Tushare Pro API (付费版)
- 计算库：Pandas, NumPy, Matplotlib
- 存储层：内存+文件系统

## 完成的具体工作

### 🎯 开发实现的核心功能

#### **1. Multi-Agent架构设计与实现** ⭐⭐⭐⭐⭐

**工作内容**：
- 设计5个协作智能体（Supervisor/Coder/Reviewer/ErrorHandler/ProfileUpdater）
- 实现LangGraph消息流编排
- 建立节点间的条件路由逻辑

**解决的关键问题**：
- ❌ **问题1**：多轮Agent调用导致消息结构混乱（ToolMessage + AIMessage排列错误）
  - ✅ **解决方案**：在Reviewer中强制将ToolMessage转换为HumanMessage，确保消息结构合法
  - **代码位置**：`agent_gradio.py` 第1090-1097行
  
- ❌ **问题2**：多次重试导致上下文爆炸
  - ✅ **解决方案**：实现消息修剪机制，只保留用户问题 + 最新数据
  - **代码位置**：`multi_agent.py` 第1055-1116行（上下文清洗逻辑）

---

#### **2. 流式输出与Chain of Thought可视化** ⭐⭐⭐⭐

**工作内容**：
- 将同步的Agent执行改为异步生成器（Generator）
- 使用Gradio的Accordion组件实现思考过程折叠
- 三分输出（报告、思考过程、用户画像）

**解决的关键问题**：
- ❌ **问题**：前端转圈圈卡顿，无法实时显示进度
  - ✅ **解决方案**：改用stream_mode="updates"逐节点推送，chat()函数返回Generator
  - **代码位置**：`agent_gradio.py` 第73-146行
  
- ❌ **问题**：最终报告混在思考过程中，难以阅读
  - ✅ **解决方案**：使用_clean_report()提取最终结论，用Accordion分离展示
  - **代码位置**：`agent_gradio.py` 第62-71行、第190-196行

---

#### **3. 数据防护与自救机制** ⭐⭐⭐⭐⭐

**工作内容**：
- 在CODER_SYSTEM_PROMPT中明确规范数据质检流程
- 实现三层自救方案（扩大范围、放宽条件、换个API）
- 添加元数据透传（[META]标签输出）

**解决的关键问题**：
- ❌ **问题1**：沪深300数据获取返回6000支而非300支
  - ✅ **解决方案**：在Prompt中明确要求`limit=300`参数
  - **代码位置**：`multi_agent.py` 第844行（API最佳实践部分）

- ❌ **问题2**：财报数据滞后2年（只有2023年年报）
  - ✅ **解决方案**：修改Prompt要求优先查最新季报而非年报
  - **代码位置**：`multi_agent.py` 第805-820行（财务数据时效性规则）

- ❌ **问题3**：DataFrame过滤逻辑错误导致JSON序列化失败
  - ✅ **解决方案**：在Prompt中提供正确的DataFrame过滤示例和类型转换方法
  - **代码位置**：`multi_agent.py` 第861-889行（JSON序列化规范）

- ❌ **问题4**：API限流（批量获取500支股票导致1000+调用）
  - ✅ **解决方案**：在Prompt中明确禁止嵌套循环，建议扩大日期范围而非逐个查询
  - **代码位置**：`multi_agent.py` 第751-756行（API调用优化）

---

#### **4. 错误分类与自动重试机制** ⭐⭐⭐⭐

**工作内容**：
- 实现4类错误分类（code_error/network_error/api_error/unknown）
- 错误自动重试（最多3次）
- 降级处理（从smart模型降级至fast模型）

**解决的关键问题**：
- ❌ **问题1**：Reviewer调用3次失败后系统卡死
  - ✅ **解决方案**：添加max_retries循环+强制降级回复（HumanMessage兜底）

**改进前**：Reviewer失败3次后返回AIMessage
```python
return {
    "messages": [AIMessage(content=fallback_content)],  # ❌ 错误
    "execution_status": "error"
}
```
问题：AIMessage带着没有配套的tool_calls，导致消息结构混乱，触发LangGraph API 400错误

**改进后**：改为返回HumanMessage
```python
return {
    "messages": [HumanMessage(content=fallback_content)],  # ✅ 正确
    "execution_status": "error"
}
```
原因：HumanMessage是"用户的话"，不会被误认为需要tool_calls，消息结构永远合法
  - **代码位置**：`multi_agent.py` 第1120-1192行

- ❌ **问题2**：无法区分"真正的失败"和"暂时的数据不足"
  - ✅ **解决方案**：使用ErrorHandler节点分类，决定是否重规划任务
  - **代码位置**：`multi_agent.py` 第1199-1350行

---

#### **5. 用户画像学习与更新** ⭐⭐⭐

**工作内容**：
- 从每次分析报告中自动提取用户风险偏好、投资风格等
- 实时更新user_profile
- 支持后续任务的个性化定制

**解决的关键问题**：
- ❌ **问题**：系统无法记住用户偏好，每次都从零开始
  - ✅ **解决方案**：ProfileUpdater节点在每次Reviewer完成后触发，自动更新画像
  - **代码位置**：`multi_agent.py` 第1357-1450行

---

#### **6. 元数据透传与Reviewer准确性改进** ⭐⭐⭐⭐

**工作内容**（最新改进）：
- Coder在输出[DATA]前必须输出[META]元数据
- 要求Reviewer在报告中引用[META]而不是瞎猜
- 确保权重配置、指标定义、数据周期100%准确

**解决的关键问题**：
- ❌ **问题1**：Reviewer在报告中瞎猜权重，导致解释与代码不符
  - ✅ **解决方案**：在CODER_SYSTEM_PROMPT中强制输出[META]标签
  - **代码位置**：`multi_agent.py` 第644-694行

- ❌ **问题2**：Reviewer不清楚指标的定义和单位
  - ✅ **解决方案**：在REVIEWER_SYSTEM_PROMPT中要求必须引用[META]中的定义
  - **代码位置**：`multi_agent.py` 第1027-1048行

### 📍 关键路由决策点

#### **Supervisor → Coder vs Reviewer**

| 条件 | 路由目标 | 理由 |
|------|---------|------|
| 用户问题涉及"数据获取"、"计算"、"绘图" | Coder | 需要代码执行 |
| Coder已输出[DATA]且无[ERROR] | Reviewer | 数据已就位，进行分析 |
| Reviewer已完成报告 | FINISH | 任务完成 |
| Coder失败3次 | ErrorHandler→FINISH | 启用降级策略 |

#### **Coder内部自救流程**

```
API调用 → 返回空数据？
    │
    ├─→ 方案A：扩大日期范围 → 成功？→ 继续
    │
    ├─→ 方案B：放宽筛选条件 → 成功？→ 继续
    │
    ├─→ 方案C：换个API接口 → 成功？→ 继续
    │
    └─→ 全部失败 → 返回[ERROR]标记 → ErrorHandler
```

## 支持的输入类型和输出形式

### 📥 输入类型

#### **支持的Prompt形式**

| 输入类型 | 示例 | 系统处理方式 |
|---------|------|-----------|
| **简单数据查询** | "查询平安银行的最新PE和股息率" | Coder直接API调用，快速返回数据 |
| **对比分析** | "对比三支银行股的估值水平" | Coder批量查询，计算对比指标 |

| **多轮问题** | 在报告基础上追问新问题 | 系统记住用户画像，提供个性化回答 |

- ✅ 技术面分析
- ✅ 择股策略回测
- ✅ 风险管理评估

- ❌ 实时行情预测（系统仅基于历史数据）
