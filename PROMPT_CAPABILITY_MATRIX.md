# 📊 多智能体系统(Multi-Agent v2.1) - Prompt能力分层矩阵

## 🎯 项目概述

**系统名称**：A股数据分析助手 v2.1 (LangGraph增强版)

**核心架构**：多智能体 + Supervisor路由 + Self-Correction机制

**关键组件**：
- 🧭 **Supervisor主管**：任务分解、智能路由
- 💻 **Coder编码员**：Python代码执行、Tushare API调用
- 📝 **Reviewer分析师**：金融报告撰写、分析结论
- 👤 **ProfileUpdater**：用户画像学习与更新
- 🔧 **ErrorHandler**：错误分类与自我修正（最多3次重试）

---

## 📌 系统能力边界

### ✅ 支持的能力
- Tushare API数据获取（日线、财务、指数成分股等）
- Python数据分析和计算（pandas、numpy）
- 财务指标计算（PE、PB、股息率、波动率、收益率等）
- 专业金融分析报告撰写
- 用户画像动态学习
- 多轮对话历史记忆

### ❌ 限制和已知问题
- 📍 **沪深300数据获取错误**：获取超过300支数据，需要加`limit=300`
- ⏰ **财报时效性问题**：只有2023年完整年报，2024+数据需用TTM/季报
- 🌐 **数据源单一**：仅支持Tushare，无其他数据源集成
- 💬 **多轮交互不稳定**：L5级多轮反馈机制成功率<50%
- 🎯 **概念分类缺失**：无法自动识别"AI芯片"等概念板块

### **L5级 - 超高级（多轮交互+动态反馈）**

**难度等级**：⭐⭐⭐⭐⭐  
**成功率**：40%-50% 🔴  
**特点**：需要4-5轮以上交互，用户实时反馈，系统动态调整  
**推荐**：❌ 不推荐生产使用（不稳定）

#### L5.1 - 多轮对话学习（有限支持）

```
第1轮用户：请分析贵州茅台，我是保守型投资者
系统 → 输出报告，更新用户画像为"保守+白酒关注"

用户反馈：我其实对消费股更感兴趣，请重新筛选
第2轮 → 系统应该记住"保守+消费"，重新筛选

用户反馈：我看好的不是白酒，是食品和日用品
第3轮 → 系统应该跳过酒类，聚焦食品日用品
```

**问题**：
- ProfileUpdater会更新画像，但不主动触发重新筛选
- 需要用户在每轮明确说明新需求

#### L5.2 - 实时市场反应（不支持）

```
当前是2025年11月，市场在炒"AI芯片"题材。
请帮我分析AI芯片公司的估值...
```

**问题**：
- Tushare无"概念"标签，无法自动识别板块
- 需要手动输入"包括以下企业：xxx, yyy, zzz..."

#### L5.3 - 组合动态调整（不支持）

```
我之前买入了【组合A】，最近表现如下...
请告诉我是否需要调整？怎么调整？
```

**问题**：
- 系统无法读取用户真实持仓
- 只能基于假设数据做分析
- 需要用户提供持仓清单

## 🎯 使用建议

### ✅ 推荐使用场景
```
日常使用：L1-L2级（快速、稳定）
 ├─ 查询股票数据（L1）
 ├─ 快速对比分析（L2）
 └─ 实时行情查询（L1）

深度分析：L3级（可靠，偶尔失败）
 ├─ 行业景气度评估
 ├─ 股票择优配置
 └─ 主题投资分析

演示展示：L4-L4.5级（演示系统能力，但可能踩坑）
 ├─ 完整财务分析
 ├─ 高股息股票筛选（推荐L4.5）
 └─ 风险管理回测
```

### ❌ 不推荐使用场景
```
生产环境：避免L5级（太不稳定，<50%成功率）
 ├─ 多轮反馈优化
 ├─ 组合动态调整
 └─ 实时交易决策
```

### 📋 Prompt撰写最佳实践

**DO（✅ 推荐）**：
- 明确指定股票代码和数据范围（"沪深300"、"过去3个月"）
- 指定要求的输出格式（"数据表格"、"评分排名"）
- 包含业务背景（"我是保守型投资者"）
- 要求具体建议而非仅数据（"给出Top 2推荐"）

**DON'T（❌ 避免）**：
- 使用模糊概念（"热门股票"、"AI概念"）- Tushare无法识别
- 要求超过1年的完整年报 - 只有2023年可用
- 一次性要求太多未来预测 - 系统不支持
- 多轮对话期间完全改变需求 - 会导致失败

### 🔧 调试技巧

**如果任务失败**：
1. 检查是否是L4+级任务（容易失败）
2. 查看日志中的`[WARNING]`和`[ERROR]`标记
3. 使用"简化版Prompt"重试（删除复杂条件）
4. 检查是否涉及"概念分类"或"超过1年年报"等已知限制
