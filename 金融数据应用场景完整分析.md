## 第三部分：具体应用场景 + 对应Prompt设计

### 📍 场景1：个人投资者 - 选股与配置（L1-L2难度）

#### **背景**
小李是一个上班族，有100万闲钱想投资。他不是专业投资者，但想做出理性决策。

#### **真实业务需求**

```
小李的核心问题：
❓ "我的风险承受能力是中等，想要每年4-5%的分红收益。
    请帮我从2000支A股中筛选出最适合的15支股票。"

这涉及多个维度：
① 风险承受能力匹配（中等=波动率10-20%）
② 收益目标匹配（4-5%的股息率）
③ 市值和流动性要求（不要买太小众的股票）
④ 行业配置（不要重仓某个行业）
⑤ 个人偏好（如排除某些行业或概念）
```

#### **对应Prompt设计**

```
【Prompt 1.1 - 筛选】
---
我有以下投资需求，请帮我筛选适合的股票：

【投资者画像】
- 风险承受能力：中等（可接受15-20%的年波动率）
- 目标收益率：每年4-5%的分红
- 投资期限：5年以上（长期）
- 投资金额：100万元

【选股条件】
1. 股息率≥4%且≤8%（过高说明有问题）
2. PE倍数：10-30倍（合理区间）
3. PB倍数：0.8-2.0倍（不要过度破净或溢价）
4. 过去3年分红稳定性：年度分红年增长率≥0%（不能下降）
5. 市值≥100亿元（流动性充足）
6. 排除：ST股、暂停上市、最近有减持公告的股票

【行业配置要求】
- 金融行业不超过30%
- 地产行业不超过10%
- 单支股票不超过10%

【最后给出】
- 15支股票清单
- 每支股票的配置比例和理由
- 预期组合年收益率和风险提示

---
```

#### **系统需要做什么**

```
【Coder需要执行】
1. 从daily_basic表获取所有股票的PE、PB、dv_ttm
2. 从dividend表获取分红历史（判断稳定性）
3. 从stock_basic表获取行业分类和市值
4. 过滤满足条件的股票
5. 计算行业配置占比
6. 输出配置清单和权重

【Reviewer需要审视】
1. 15支股票是否真的满足所有条件
2. 配置权重是否合理（总和=100%）
3. 分红稳定性分析是否准确
4. 风险提示是否充分（如"股息率高可能意味着公司现金流压力"）

【ProfileUpdater需要记住】
- 这个用户风险承受能力"中等"
- 偏好"高分红"策略
- 投资期限"长期"
- 下次推荐时，自动筛选符合这些特征的策略
```

#### **系统能提供的价值**

```
传统方式：
1. 打开股票软件，手工查看2000支股票的PE和分红
2. 逐个看，逐个筛选（需要4-6小时）
3. 手工计算配置权重
4. 写个Word文档总结

AI系统：
1. 用Prompt描述需求
2. 3分钟获得精准的15支股票清单
3. 自动计算配置权重
4. 自动输出专业的分析报告

效率提升：系统做人工4小时的工作，用3分钟完成
价值：小李可以立即开始投资，而不是花一天时间准备
```

### 📍 场景2：风险投资人 - 项目估值与对标分析（L2-L3难度）

#### **背景**
王总是一个PE投资人，要评估一家拟上市的科技公司。需要用同行对标来评估这家公司的合理估值。

#### **真实业务需求**

```
王总的核心问题：
❓ "我有一个科技公司要评估，他们说应该用50倍PE定价。
    我想知道：
    (1) 同行业的平均PE是多少？
    (2) 如果按现在的赛道平均PE，应该估值多少？
    (3) 这家公司的增速有没有特别的地方？"

这涉及：
① 同行业选择（定义"同行"）
② 财务指标快速查询（收入、利润、增长率）
③ 估值倍数对标（PE、PEG、PS）
④ 多维估值方法（DCF、相对估值、可比公司法）
```

#### **对应Prompt设计**

```
【Prompt 2.1 - 对标分析】
---
请进行以下公司的可比公司法估值分析：

【目标公司】
名称：某科技公司（未上市）
2024年预计收入：10亿元
2024年预计净利润：1.5亿元
过去3年利润复合增长率：45%（高速增长）
主营业务：云计算/SaaS解决方案

【对标要求】
1. 从A股上市公司中选出5-8家"同类型"的科技公司
   判断同类型的标准：
   - 主营业务相似（云计算/SaaS）
   - 市值相当或略大（100亿-1000亿）
   - 上市满2年以上

2. 对每家对标公司收集以下数据：
   - 最新PE（TTM）、PEG（基于未来增速）、PS
   - 过去3年利润复合增长率
   - 毛利率和净利率

3. 计算对标公司的：
   - PE平均值、中位数
   - PEG平均值、中位数
   - PS平均值、中位数

4. 基于对标给目标公司定价：
   - 如果用平均PE：合理估值 = 1.5亿 × 平均PE
   - 如果用中位数PE：合理估值 = 1.5亿 × 中位数PE
   - 如果用PEG方法：估值 = 1.5亿 × (PE × 增速)

5. 最后给出：
   - "这家公司的合理估值范围是XX亿-YY亿"
   - "与市场报价50倍PE相比，是高估还是低估"
   - "定价时要注意的风险因素"

---
```

#### **系统需要做什么**

```
【难点1：找到"同类型"公司】
✓ Coder需要：
  - 从stock_basic获取所有科技行业的公司
  - 用关键词匹配（云计算、SaaS等）
  - 按市值筛选（100亿-1000亿）
  - 获取上市年份，排除<2年的

✓ 这涉及"概念分类"——Tushare本身没有
  → 需要用"行业代码"和"名称关键词"智能匹配
  → 这是系统需要改进的地方

【难点2：计算增速和PEG】
✓ Coder需要：
  - 从income表获取过去3年的净利润
  - 计算CAGR（复合年增长率）
  - 计算PEG = PE / 增长率

【难点3：做出估值结论】
✓ Reviewer需要：
  - 理解"为什么PEG在15倍以下是合理的"
  - 理解"增速越高，能承受的PE越高"
  - 给出定价建议（而不仅仅是数据）

【难点4：自我意识】
✓ ErrorHandler需要：
  - 如果找到的"同类型"公司太少（<3家）
    → 自动降级：拓宽行业定义，增加IT类的所有公司
  - 如果数据不完整
    → 自动找替代数据源（如TTM而非单年）
```

#### **系统能提供的价值**

```
传统方式（PE投资人的标准流程）：
1. 在Wind、Bloomberg上查询"云计算概念股"（需要花钱买软件）
2. 逐个点开查看PE、增速（1-2小时）
3. 手工计算PEG
4. 在Excel里做对标分析
5. 写估值报告
总耗时：2-3小时

AI系统：
1. 输入一个Prompt
2. 系统自动找对标公司
3. 自动计算PE、PEG、PS
4. 自动给出估值范围
5. 输出专业报告
总耗时：5分钟

价值：
- 没有Wind账户的小机构也能做对标分析
- 可以快速验证投资机会
- 在投委会上有数据支撑，而不是主观判断
```

### 📍 场景3：基金经理 - 组合风险管理（L3-L4难度）

#### **背景**
李女士管理一个50亿的混合基金，需要每天监控组合的风险指标。

#### **真实业务需求**

```
李女士的核心问题：
❓ "我的投资组合有30支股票，总额50亿。
    我需要每天知道：
    (1) 组合的整体风险水平（用波动率衡量）
    (2) 是否有股票"异常"了（价格涨跌超过历史范围）
    (3) 相关性分析（哪些股票容易一起跌）
    (4) 压力测试（如果大盘跌10%，我的组合会跌多少）"

这涉及：
① 实时数据监控
② 风险指标计算（VaR、波动率、贝塔值）
③ 相关性分析（主成分分析）
④ 压力测试和情景分析
```

#### **对应Prompt设计**

```
【Prompt 3.1 - 组合风险监控】
---
请对我的投资组合进行风险评估：

【组合持仓】
股票1：贵州茅台 - 5亿元（10%）
股票2：中国平安 - 2亿元（4%）
股票3：招商银行 - 3亿元（6%）
... 再加27支股票
总规模：50亿元

【监控周期】
数据：最近3个月的日线数据（近60个交易日）

【需要输出】
1. 组合的基本风险指标
   - 组合日均波动率
   - 组合的年化波动率
   - 与沪深300的相关性（贝塔值）

2. 个股异常预警
   - 哪些个股的波动率突然升高？
   - 哪些个股的涨跌趋势与大盘反向？
   - 哪些个股存在"黑天鹅"（单日涨跌>5%）

3. 相关性热力图
   - 30支股票两两之间的相关系数
   - 找出相关性最高的股票对（容易一起跌）
   - 找出相关性最低/为负的股票对（可以互相对冲）

4. 压力测试
   - 假设沪深300下跌10%，我的组合预期下跌多少？
   - 假设下跌20%，预期下跌多少？
   - 假设下跌30%（极端场景），预期下跌多少？

5. 建议
   - 当前组合的风险水平：高/中/低
   - 如果要降低风险，应该卖出哪些相关性高的股票
   - 如果要增加防守，应该加配哪些防守品种

---
```

#### **系统需要做什么**

```
【核心难点1：计算相关性矩阵】
✓ 这不仅仅是"查数据"，而是"复杂计算"
  需要：
  - 获取30支股票的日线数据（60交易日 × 30支）
  - 计算日收益率（30 × 60的矩阵）
  - 计算两两相关系数（30×30的矩阵，900个相关系数）
  - 这需要numpy/scipy的矩阵运算

✓ Coder需要：
  import numpy as np
  import pandas as pd
  # 获取日线数据
  returns = df[['close']].pct_change()
  # 计算相关矩阵
  corr_matrix = returns.corr()

【核心难点2：计算贝塔值】
✓ 贝塔值 = 组合与指数的协方差 / 指数的方差
  需要：
  - 获取沪深300的日线数据
  - 计算组合收益率（加权平均30支股票）
  - 计算组合和指数的协方差

【核心难点3：压力测试】
✓ 假设大盘跌10%，组合会跌多少？
  计算逻辑：
  - 大盘跌幅 × 组合的贝塔值 = 组合预期跌幅
  - 例如：大盘跌10% × 组合贝塔1.2 = 组合跌12%

【核心难点4：Reviewer的金融素养】
✓ 不仅要输出"相关系数是0.85"
  而是要说："贵州茅台和中国平安的相关系数是0.85，
            说明这两支股票走势高度一致，无法起到分散风险的作用"

✓ 要能给出"降低风险的建议"：
  "建议卖出与组合整体走势高度一致的股票，
   加配一些与消费股相关性低的防守品种（如公用事业）"
```

#### **系统能提供的价值**

```
传统方式（基金经理的标准流程）：
1. 打开Wind软件
2. 逐支输入股票，查看数据
3. 用Excel做相关性分析（VBA或Python脚本）
4. 生成风险报告
总耗时：每天1-2小时

AI系统：
1. 输入组合持仓和权重
2. 系统自动获取数据
3. 自动计算相关性、贝塔值
4. 自动进行压力测试
5. 给出风险预警和优化建议
总耗时：5-10分钟

价值：
- 实时监控，而不是定期报告
- 更多时间做"投资决策"而不是"数据收集"
- 可以快速验证新的配置想法
```

### 📍 场景4：财务分析师 - 企业基本面研究（L2-L3难度）

#### **背景**
张先生是券商的行业研究员，要写一份"地产行业年度展望"。

#### **真实业务需求**

```
张先生的核心问题：
❓ "我要写个关于房地产行业的深度报告。
    需要：
    (1) 对比过去3年，地产行业的营收和利润趋势
    (2) 主要房企（万科、保利、碧桂园等）的财务对标
    (3) 地产行业的估值变化（PE、PB如何演进）
    (4) 给出对2025年地产行业的判断"

这涉及：
① 财务数据聚合（行业层面）
② 企业对标分析
③ 估值历史追踪
④ 行业趋势判断
```

#### **对应Prompt设计**

```
【Prompt 4.1 - 行业深度分析】
---
请帮我做一份房地产行业的深度研究：

【背景】
我要给客户写一份"2025年房地产行业展望"报告

【需要的分析内容】
1. 行业规模和增长
   - 统计"房地产开发"行业的所有上市公司
   - 计算过去3年这个行业的总收入、总利润、增长率
   - 对比医药、消费等其他行业的增速

2. 主要企业对标
   - 列出市值前10的房地产企业
   - 对每家企业，统计：
     - 最近3年的营收、利润、增长率
     - 当前的PE、PB、股息率
     - ROE（衡量赚钱效率）
   - 做成对比表

3. 行业估值历史追踪
   - 过去5年，房地产行业的平均PE如何变化？
   - 现在的PE与历史对比，是高估还是低估？
   - 与医药、消费等防守品种的PE对比

4. 定性分析
   - 列出房地产行业面临的主要风险：
     a) 政策风险（调控）
     b) 需求风险（人口、城市化）
     c) 财务风险（债务、现金流）
   - 列出可能的利好因素
   - 对2025年房地产的前景评判

5. 配置建议
   - 当前是否应该配置房地产？多少比例？
   - 如果要配置，应该选大公司还是小公司？
   - 与去年相比，今年的配置建议有什么改变？

---
```

#### **系统需要做什么**

```
【难点1：行业数据聚合】
✓ "房地产行业"在Tushare中是什么？
  - 可能是"房地产业"（行业分类）
  - 可能是"房地产开发"（子行业）
  
✓ Coder需要：
  - 从stock_basic获取所有"房地产"企业的列表
  - 从income/balance表获取营收、利润数据
  - 从daily_basic获取PE、PB
  - 聚合计算行业总体指标

【难点2：历史估值追踪】
✓ 不是"现在的PE是多少"
  而是"过去5年，房地产行业的PE如何演变"
  
✓ 这需要：
  - 获取过去5年的历史PE数据（daily_basic有历史数据）
  - 按行业汇总
  - 画出PE随时间的变化曲线

【难点3：Reviewer的行业判断】
✓ 不能只输出"房地产PE从8倍升到12倍"
  还要说："PE升高说明市场对房地产前景更乐观"
           或者"可能是因为利率下降"
           或者"可能是由于预期改善"

✓ 这需要Reviewer有"行业背景知识"
  - 理解房地产周期
  - 理解政策的影响
  - 理解人口和城市化的关系

【难点4：多源数据整合】
✓ 这个分析可能需要：
  - 股票数据（Tushare）
  - 政策新闻（需要额外数据源）
  - 宏观数据如利率、房价指数（需要额外数据源）

✓ 目前系统只有Tushare，无法完全支持
  → 可以说"基于股票市场反映的信息做分析"
  → 但不能做"完全的行业预测"
```

#### **系统能提供的价值**

```
传统方式（研究员的标准流程）：
1. 收集行业数据（Wind、国际商业机器公司等付费数据库）
2. 查找企业财报（逐个看上市公司的Q1/Q2/Q3/Q4财报）
3. 在Excel做对标分析
4. 撰写研究报告（3000-5000字）
总耗时：2-3周

AI系统：
1. 输入行业名称和分析维度
2. 系统自动收集数据、做对标、做历史追踪

总耗时：系统5-10分钟做基础工作，研究员1-2天做定性分析

价值：
- 加快基础数据收集
- 更多时间做深度思考
- 可以快速更新行业观点（当有重大事件发生时）
```

## 第七部分：总结对标表

### 📊 系统当前覆盖的场景 vs 未覆盖的场景

```
【✅ 已经覆盖的场景】

个人投资者
├─ ✓ 单支股票分析（"这支股票值不值得买？"）
├─ ✓ 多支股票对比（"A和B哪个更好？"）
└─ ✓ 选股筛选（"帮我从N支股票中筛出最好的"）

机构投资者（股票研究）
├─ ✓ 财务对标（"对比5家同行，谁的PE最低？"）
├─ ✓ 估值定价（"这个项目应该怎么估值？"）
└─ ✓ 简单的行业对比（"3个月的行业平均PE"）

【❌ 未覆盖的场景】

基金经理 - 组合风险管理
├─ ❌ 实时监控（系统是"查一次"，不是"持续监控"）
├─ ❌ 高频预警（需要自动化的消息推送）
└─ ❌ 动态对冲（需要实盘交易接口）

行业研究员 - 深度报告
├─ ❌ 宏观关联分析（"利率与PE的关系"）
├─ ❌ 政策影响评估
└─ ❌ 多年份历史对比（目前只能查最新数据）

量化交易员
├─ ❌ 量化选股模型
├─ ❌ 因子分析
└─ ❌ 回测框架

风险管理部门
├─ ❌ 实时风险监控
├─ ❌ VaR预测
└─ ❌ 压力测试（动态的，而非静态的）

企业融资部
├─ ❌ 融资估值建议
├─ ❌ 可比公司法完整流程
└─ ❌ 并购重组定价

财务部门
├─ ❌ 财务预测（基于历史，推断未来）
├─ ❌ 成本控制建议
└─ ❌ 现金流管理优化
```

## 最终建议

### 🎯 系统的定位应该是

```
【现阶段】
"为个人投资者和初级研究员提供快速的股票分析和筛选工具"
═══════════════════════════════════════════════════════

能做好的事：
✓ 查询股票数据
✓ 计算财务指标
✓ 做估值对标
✓ 写分析报告

能做但不完美的事：
⚠️ 行业分析（因为缺少概念分类和历史数据）
⚠️ 风险管理（因为缺少实时监控）
⚠️ 趋势预测（因为缺少宏观数据）

做不了的事：
❌ 实时监控和预警
❌ 多源数据集成
❌ 高频交易
❌ 完整的投资决策支持（缺少主观判断部分）

【未来方向】
如果要做"完整的机构级投资决策系统"，需要：
1. 接入更多数据源（宏观、行业、舆情等）
2. 添加实时监控和消息推送
3. 支持交易接口（至少是仿真）
4. 强化Reviewer的行业知识
5. 添加多智能体协作（分析师+风险官+交易员）
```
