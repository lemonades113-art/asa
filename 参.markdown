# 📝 A股多智能体交易系统 - 项目核心成果汇编
**（供报告撰写与PPT制作使用）**

## 1. 项目核心概览 (Executive Summary)

*   **项目名称**：基于 LangGraph 的 A股多智能体（Multi-Agent）投研代理系统
*   **一句话介绍**：这不是一个简单的聊天机器人，而是一个由5个AI智能体组成的**“虚拟投研团队”**。它能像初级分析师一样，自主编写代码获取实时数据、进行自我纠错、并生成有理有据的投资分析报告。
*   **核心技术栈**：LangGraph (编排), qwen-plus/qwen-turbo (大脑), Tushare (数据源), Python (执行环境)。

---

## 2. 为什么要不仅做这个？(Motivation)

**核心痛点：金融AI的“不可能三角”**
*(建议在报告/PPT开头强调此背景)*

现有通用大模型（如ChatGPT）在金融领域存在三个致命缺陷：
1.  **数据滞后**：训练数据截止以前，不知道今天的股价（无法做实时分析）。
2.  **计算幻觉**：简单的加减乘除常出错，无法进行复杂的金融计算（如夏普比率）。
3.  **过程黑盒**：给出的结论无法验证，没有数据来源，不可信。

**我们的解决方案**：
通过 **Agentic Workflow（代理工作流）**，让AI学会“使用工具”和“自我反思”，实现：
*   ✅ **实时性**：直接调用 API 获取最新行情。
*   ✅ **准确性**：通过 Python 代码计算，而非语言模型推算。
*   ✅ **透明性**：每一句结论都带有数据来源和计算逻辑。

## 3. 系统架构设计 (System Architecture)

*(适合放在“方法论”或“系统设计”章节)*

我们设计了一个**Supervisor-Worker（星形拓扑）**架构，包含五大职能角色：

| 角色 (Agent) | 拟人化职能 | 核心任务 | 技术亮点 |
| :--- | :--- | :--- | :--- |
| **Supervisor** | 🧠 大脑/组长 | 拆解任务、分发工单 | 动态路由，避免死锁 |
| **Coder** | ✋ 手脚/程序员 | 写代码、查数据 | 状态持久化，强制元数据输出 |
| **Reviewer** | 👁️ 眼睛/质椝员 | 检查异常值、写报告 | 拦截 >10% 股息率等异常数据 |
| **ErrorHandler** | 🛡️ 免疫系统 | 自动修复报错 | 代码自动重写，指数退避机制 |
| **ProfileUpdater** | 💾 记忆中枢 |  | 多轮会话记忆（如记住你是激进型投资者） |

## 4. 三大核心创新点 (Key Innovations)

*(这是项目的加分项，必须重点突出)*

### 🌟 创新一：四级自我修复机制 (Self-Correction)
*   **论点**：系统具有“反脆弱性”，出错不会崩溃，而是会自动修复。
*   **证据**：
    *   **代码修复**：当 `pd.Series.to_json()` 报错时，ErrorHandler 自动改为 `json.dumps()`，重试成功率 95%。
    *   **数据补救**：当缺少“应收账款”字段时，系统自动降级使用代理变量（Fallback机制），保证流程不中断。
    *   **逻辑纠正**：当算出 515% 的异常股息率时，Reviewer 会识别并要求重算。

### 🌟 创新二：全链路数据溯源 (Traceability)
*   **论点**：拒绝幻觉，每一条结论都有据可查。
*   **证据**：强制输出四层标签 `[DATE]`(时间)、`[SOURCE]`(来源)、`[META]`(计算公式)、`[DATA]`(原始数据)。
*   **效果**：用户得到的不是模糊建议，而是类似“基于 Tushare 11月30日数据，根据 PE=股价/EPS 公式计算...”的严谨报告。

### 🌟 创新三：模型分层策略 (Model Layering)
*   **论点**：好钢用在刀刃上，平衡性能与成本。
*   **证据**：
    *   **Smart Model (qwen-plus)**：负责写代码、做决策（占 30% 调用）。
    *   **Fast Model (qwen-turbo)**：负责纠错、记忆提取（占 70% 调用）。
*   **效果**：Token 成本降低 30%，且并未牺牲核心任务的准确性。

## 5. 真实能力演示 (Case Studies)

*(报告中最具说服力的部分，建议选用以下三个典型案例)*

### 案例 1：海康威视——财务排雷（展示“数据诚实性”）
*   **任务**：分析是否有赊销冲业绩。
*   **过程**：第一次执行发现“应收账款”字段缺失 → 触发 Fallback 机制 → 使用“总资产”相关项做代理。
*   **结论**：2024年应收账款占比畸高（297.5%），净现比低（0.944），系统提示存在“赊销冲业绩”嫌疑。
*   **亮点**：系统诚实地标记了 `[WARNING]`，告知用户使用了替代数据，而非编造数据。

### 案例 2：中国神华——风险量化（展示“自我修复力”）
*   **任务**：计算夏普比率、最大回撤。
*   **过程**：代码运行报错（类型错误） → ErrorHandler 介入 → **自动修改代码** → 第2次运行成功。
*   **结论**：年化波动率 23.10%，最大回撤 -19.05%。
*   **亮点**：用户无感知的情况下，系统在后台完成了错误修复。

### 案例 3：中信证券——技术面分析（展示“多维分析力”）
*   **任务**：MACD + 布林带分析。
*   **结论**：自动识别金叉日期（09-29），当前为空头排列，建议观望。
*   **亮点**：能够结合价格、成交量、指标三个维度给出综合建议，而非单一指标。

## 6. 局限性与未来展望 (Limitations)

*(写在报告最后，显示客观严谨的学术态度)*

尽管系统有输出，但目前仍存在以下局限：
1.  **响应速度**：平均耗时 15-45 秒，不适合高频交易（未来优化方向：并行计算）。
2.  **多轮对话记忆**：虽然有记忆模块，但缺乏“数据缓存”，也会上下文管理的问题，问完茅台再问五粮液，可能需要重新拉取数据（未来优化方向：Redis 缓存层）。
3.  **视觉能力缺失**：无法直接看懂用户上传的 K 线图图片（未来优化方向：接入多模态大模型）。
