## 🎯 三种测试方案

### 方案1️⃣: 快速验证 (10分钟) ⚡
适合初次上手或快速检验系统状态

```
L1.1 (1分钟)   →  L2.1 (3分钟)  →  L3.1 (5分钟)  →ℹ️ 观察结果
```

流程：热身 → 简单组合 → 中等复杂

---

### 方案2️⃣: 标准测试 (30分钟) 🔍
适合系统开发和优化，全面了解系统能力

```
L1 (3分钟)  →  L2 (8分钟)  →  L3 (10分钟)  →  L4 (9分钟)
```

流程：
1. L1.1, L1.2 → 验证基础
2. L2.1, L2.2 → 验证组合能力
3. L3.1, L3.2 → 验证复杂编排
4. L4.1, L4.2 → 验证容错恢复

---

### 方案3️⃣: 完整测试 (90分钟) 🚀
适合上线前的完整验收，包括极限场景

```
【基础】L1 (3分钟)
  ↓
【简单】L2 (8分钟)
  ↓
【中等】L3 (10分钟)
  ↓
【困难】L4 (15分钟)
  ↓
【极限】L5.1 (20分钟) + L5.2 (25分钟)
  ↓
【总结】分析能力边界
```
