# 如何跟演讲同学沟通 - 完整指南

> **你的角色**：项目开发者，负责代码和技术  
> **他的角色**：演讲人，负责讲故事和回答问题  
> **目标**：他讲得清楚自信，不露馅，你不用担心他不懂代码

---

## 第一步：第一次见面时怎么说

### 开场白（5分钟）

```
"我们项目这次需要分工，我负责开发，你负责演讲。
这是很正常的安排——就像一部电影，导演和演员是不同的人。

我把整个系统的逻辑都整理成文档了。
你不需要理解每一行代码怎么写的，
只需要知道'我们做了什么、能做什么、带来什么价值'。

我给你准备了三份材料：
1. 演讲脚本和逻辑框架（你要背这个）
2. 突发问题应急手册（评委可能问这些）
3. 速查表（现场查这个）

现在花 30 分钟，我给你讲一遍核心逻辑。"
```

---

## 第二步：给他讲 核心10分钟

### 讲法（用类比，不用技术词汇）

```
【架构】
"想象一个投资研究小组。有 5 个专业人士：
- Boss（Supervisor）：理解客户需求，分配任务
- 程序员（Coder）：写代码查数据
- 质检员（Reviewer）：检查结果有没有问题  
- 维修工（ErrorHandler）：出错了就修
- 秘书（ProfileUpdater）：记住客户的偏好

我们的系统就是这 5 个角色的虚拟版本。"

【流程】
"用户问'帮我查中国平安的估值'：
第1步：Boss 理解这个需求 → 分解成 4 个小任务
第2步：程序员执行代码 → 从 Tushare API 抓数据
第3步：质检员审核 → 检查数据对不对（有没有异常）
第4步：维修工检查 → 如果出错自动修
第5步：秘书记住 → 下次用户问时更快
第6步：输出报告 → 用户收到分析结果"

【创新】
"为什么这样设计？三个理由：
1. 分工清晰 → 每个角色只做自己最擅长的事
2. 质量高 → 有质检员检查，坏数据排不出去
3. 容易修 → 出问题时知道在哪个环节"

【成果】
"真实验证：跑了 44 个任务，86% 最后成功。
时间对比：手工 2-3 小时 vs AI 系统 10 分钟。
成本对比：人工成本 300 万/年 vs AI 成本 5-10 万/年。"
```

## 第三步：教他怎么讲

### 演讲的四个阶段

**🎬 第 1 分钟：开场（讲问题 + 讲定位）**

给他一个脚本：
```
"大家好。初级分析师的日常工作很重复：
查数据、算指标、写报告。一份报告 2-3 小时。

我们的项目不是找更聪明的人，而是用 AI 替代这个过程。
我们没做一个'超级 AI'，而是搭建了一个'虚拟投研团队'。
5 个专业角色，各司其职。

结果是同样的工作 10 分钟自动完成。下面我给大家演示。"
```

**🎪 第 2-3 分钟：讲架构和角色**

教他用这个顺序：
```
第1步：展示 5 个角色的图
第2步：用类比讲每个角色（"Boss 是大脑，程序员是手脚"）
第3步：讲一个完整的流程（从问题到答案）
```

**📊 第 4 分钟：讲能力演示**

4 个能力逐个讲：
```
能力 1：自动查数据（展示截图：表格）
能力 2：自动检测异常（展示截图：异常数据标记）
能力 3：自动写报告（展示截图：分析文章）
能力 4：自动修复错误（讲故事：怎么处理的）
```

**💰 第 5 分钟：讲价值 + 答问**

```
效率：12-18 倍提升（手工 2-3 小时 → AI 10 分钟）
成本：节省 97%（人工 300 万 → AI 5-10 万）
质量：有质检机制，坏数据检测率 95% 以上

然后预留时间给答问。
```

## 第四步：告诉他常见问题怎么答

### 评委可能问的 8 个问题

**问 1：代码怎么写的？**
```
演讲人答法：
"这个由开发团队负责。我们的分工是我负责演讲展示，
开发同学负责代码实现。课后可以给你讲代码细节。"
```

**问 2：为什么要 5 个 Agent？**
```
演讲人答法：
"三个理由：(1) 分工清楚，每个角色只做自己擅长的
(2) 质量高，有质检员把关
(3) 容易修，出问题时知道在哪个环节"
```

**问 3：成功率 86% 怎么算的？**
```
演讲人答法：
"这是真实系统验证的数据。
我们跑了 44 个不同的查询任务，其中 36 个最后成功，
6 个虽然第一次失败，但通过自动修复也成功了，
只有 2 个最后失败。所以最终成功率 86%。"
```

**问 4：能用来做自动交易吗？**
```
演讲人答法：
"不行。我们是日级分析系统，
关心'值不值得持有这只股票 3 个月'。
自动交易需要毫秒级反应，是完全不同的系统。"
```

**问 5：用的什么 AI 模型？**
```
演讲人答法：
"我们用了国内的大语言模型（Qwen 系列）。
选择国内模型的原因是：稳定性高、成本低、中文理解强。
具体怎么集成的是开发同学的工作。"
```

**问 6：成本多少钱？**
```
演讲人答法：
"主要三部分成本：
(1) 数据 API：5000 元/年（Tushare）
(2) AI 模型调用：万分之几元/次（非常便宜）
(3) 服务器：100-500 元/月

一家 50 人研究部用这个系统替代 10 个初级分析师，
年成本 5-10 万，对标人工成本 300 万，节省 97%。"
```

**问 7：能保证不幻觉吗？**
```
演讲人答法：
"不能 0 幻觉，但我们做到了'可检测可修复'。
(1) 数据部分：直接从 API 抓，几乎不幻觉
(2) 分析部分：有质检员检查逻辑合理性
(3) 决策部分：最终投资决策必须人工审核

我们的定位是'初级分析师的替代品'，不是'投资顾问'。"
```

**问 8：我不知道的问题**
```
演讲人答法：
"这是个很好的问题。这个细节我现在答不上来，
可以课后和开发同学一起给你详细讲解。"

（诚实比瞎编好，评委会很尊重这个态度）
```

## 第五步：演讲前 1 天的排练

### 排练清单

- [ ] 讲稿背过了（至少前 3 分钟要熟）
- [ ] 5 个角色的关键词能说出来
- [ ] 2 个案例的故事线清楚了
- [ ] 常见问题的答案过了一遍
- [ ] 3 个演示视频/截图能正常播放
- [ ] 网络和设备测试过了
- [ ] 衣着得体，精神状态好

### 模拟现场

"我们一起过一遍演讲：
- 我坐下面当评委，听你讲
- 我会随机提问，看你怎么应对
- 如果讲得不清楚，我会告诉你改怎么讲
- 最后给你的演讲打个分"

## 第六步：演讲当天的注意事项

### 给演讲同学的 5 个建议

1. **讲的时候看观众，不要看屏幕**
   - 屏幕是辅助，你的讲解才是主角
   
2. **讲故事而不是讲功能**
   - ❌ "系统有 5 个 Agent，每个 Agent 有...特点..."
   - ✅ "想象你是 Boss，你要分配工作。某个 Agent 是你的程序员..."

3. **用数字说话，用对比显示价值**
   - ❌ "系统很快"
   - ✅ "10 分钟 vs 2-3 小时，快了 12 倍"

4. **不要说你不确定的话**
   - 不知道的问题就说"课后回复"，比乱编好

5. **预留 2-3 分钟给意外**
   - 视频可能卡壳
   - 网络可能延迟
   - 提前 30 秒的备用方案

### 现场突发状况处理

**系统崩溃了怎么办？**
```
不要panic，镇定地说：
"现场网络有点问题，我给大家播放提前录制的演示视频..."
（播放 30 秒的执行视频）
"这是真实的系统运行效果。或者我给大家讲一个真实的案例..."
```

**被问超难的问题怎么办？**
```
诚实地说：
"这是个很深的技术细节，我现在不掌握。
可以课后和开发同学一起给你详细讲。
但我知道这个功能的作用是..."
（转到你知道的部分）
```

## 第七步：演讲后的反馈

### 收集反馈

演讲后问他：
- "评委问了什么问题？你怎么答的？"
- "哪个部分讲得最清楚？哪个部分观众听不太懂？"
- "有什么想改进的地方吗？"

如果有被问到的问题你也没想到的答案，记下来，以后都能用。

---

## 核心原则总结

✅ **你要做**：
- 把复杂的代码逻辑简化成故事
- 提供完整的讲稿和应急方案
- 演讲前充分排练
- 现场支持他（如果评委需要技术细节）

❌ **不要做**：
- 让他看代码（太复杂，容易露馅）
- 给他太多细节（记不住，讲得乱）
- 演讲前才告诉他可能被问什么
- 演讲中不停纠正他（影响他的自信心）

---

## 给他的最后建议

**信心来自：**
1. ✅ 充分了解项目做了什么（你的讲稿）
2. ✅ 充分排练（过 1-2 遍后会很自信）
3. ✅ 知道答不上来时怎么办（应急手册）
4. ✅ 背后有你支持（课后的技术问题你来答）

**他不需要：**
1. ❌ 理解每一行代码
2. ❌ 回答很深的技术问题
3. ❌ 一个人承担所有压力

**一句话**：他是"项目的CEO"，你是"CTO"。CEO讲故事，CTO负责技术细节。分工合作，各尽其责。
