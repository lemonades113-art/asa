【问】如果某个改动出错了怎么办？
【答】
  每个改动都可以独立回滚：
  
  如果 reporter_node() 有 bug：
    → 注释掉 workflow.add_node("Reporter", reporter_node)
    → 系统仍能运行（只是没有 Reporter）
    → 快速修复后再打开
  
  如果路由逻辑有问题：
    → 改 Supervisor Prompt，让 LLM 永不派遣 Reporter
    → 系统降级回 4-Agent 模式
    → 找到问题再修改
  
  如果新闻搜索函数有问题：
    → 改 search_news_aggregated() 直接返回空结果
    → 系统仍能运行（new_data = {}）
    → 修复函数后再启用

【问】改动有什么是我需要特别注意的？
【答】
  ⚠️  4 个需要注意的地方：
  
  1️⃣  State 字段的类型
      新增字段都要在 MultiAgentState 中声明类型
      
      ❌ 错：news_data  （缺少类型）
      ✅ 对：news_data: dict
  
  2️⃣  返回值必须完整
      reporter_node() 返回的字典必须包含所有可能被读取的字段
      
      ❌ 错：return {"messages": [...]}  （漏了 last_sender）
      ✅ 对：return {
          "messages": [...],
          "news_data": {...},
          "last_sender": "Reporter",
          "reporter_ready": True
        }
  
  3️⃣  StateGraph 的连接不能漏
      每个新节点都要有入边和出边
      
      ❌ 错：add_node("Reporter", reporter_node)  （漏了连接）
      ✅ 对：
        workflow.add_node("Reporter", reporter_node)
        workflow.add_conditional_edges("Supervisor", ..., {"Reporter": "Reporter", ...})
        workflow.add_edge("Reporter", "Supervisor")
  
  4️⃣  初始化一定要做
      get_initial_state() 中要初始化所有新字段
      
      ❌ 错：新字段没有初始值
      ✅ 对：
        "news_data": {},
        "reporter_ready": False,
        ...

【问】改动完后，如果系统还是无法运行怎么办？
【答】
  【诊断步骤】：
  
  Step 1. 检查是否有 Python 语法错误
         python -m py_compile multi_agent.py
         
  Step 2. 检查 State 初始化
         print(get_initial_state())
         → 应该有 news_data, reporter_ready 等字段
  
  Step 3. 检查 StateGraph 编译
         multi_agent_app = workflow.compile(checkpointer=memory)
         → 应该成功，无 "node not found" 错误
  
  Step 4. 检查单个节点
         state = get_initial_state()
         result = reporter_node(state)
         → 应该返回完整的字典
  
  Step 5. 检查日志
         运行系统，观察控制台输出
         → 找到第一个出错的地方（通常都标有 [Reporter] 或 [Supervisor]）
