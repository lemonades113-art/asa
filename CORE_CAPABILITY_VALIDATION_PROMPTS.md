# 🔬 核心能力验证Prompt集 - 完全可运行版

> **设计目标**：每个Prompt都对应项目文档中的某个核心能力声称
> 
> **使用方法**：将以下Prompt复制到系统Web界面直接运行，观察系统的输出是否符合预期
> 
> **验证标准**：不仅看"能否运行"，更重要看"结果是否符合金融逻辑、是否包含元数据、是否能追溯"

## 第三部分：量化计算能力验证

### ✅ 能力3.1：复杂财务指标计算

**声称**：系统能精确计算复杂金融指标（波动率、相关系数、夏普比率等）

**验证Prompt**：
```
对中国神华(601088.SH)进行风险分析，要求计算：

1. 最近252个交易日（约1年）的日线数据
2. 计算每日对数收益率：ln(Pt / Pt-1)
3. 计算年化波动率：日波动率 × √252
4. 计算最大回撤：从历史高点到低点的最大跌幅
5. 计算夏普比率：(年化收益 - 2%无风险率) / 年化波动率

最后输出：
- 完整的计算过程和公式
- 四个指标的数值结果
- 财务含义解释（这些指标说明了什么？）
- 数据准确性说明（如实际用了多少条数据？）
```

**预期验证点**：
- ✅ 四个指标都有具体数值（不是模糊描述）
- ✅ 输出包含 [META] 标签，明确定义了每个指标的计算公式
- ✅ 指标值在合理范围（波动率0%-100%、夏普比率-10到+10）
- ✅ 解释了数据质量（如"实际使用了251条数据"）

---

### ✅ 能力3.2：技术指标计算（MACD、布林带）

**声称**：系统能计算技术分析指标，识别买卖信号

**验证Prompt**：
```
对中信证券(600030.SH)进行技术面分析：

1. 获取最近120个交易日的日线数据(OHLC + 成交量)

2. 计算MACD指标：
   - 快线(EMA12)和慢线(EMA26)
   - 信号线(EMA9)
   - MACD柱状值 = 快线 - 慢线
   
3. 计算布林带：
   - 中轨 = 20日简单移动平均线
   - 上轨 = 中轨 + 2 × 20日标准差
   - 下轨 = 中轨 - 2 × 20日标准差

4. 识别交易信号：
   - MACD何时出现金叉（买入信号）？
   - MACD何时出现死叉（卖出信号）？
   - 当前价格处于布林带的什么位置？
   - 结合成交量，判断当前是否该买入或卖出？

要求：
- 列出最近3个月内所有金叉和死叉的日期
- 用表格展示最近10个交易日的MACD和布林带数据
- 给出当前时刻的交易建议和风险提示
```

**预期验证点**：
- ✅ MACD和布林带数值是通过代码精确计算的（而非猜测）
- ✅ 金叉/死叉日期和数值是对应的（可以复核）
- ✅ 表格数据准确（如布林带上轨 > 中轨 > 下轨）
- ✅ 输出包含 [META] 说明了计算周期（EMA12、EMA26等）

## 第四部分：元数据透传与可验证性

### ✅ 能力4.1：完整的数据证据链【验证可解释AI】

**声称**：每个结论都能追溯到源数据和计算逻辑

**验证Prompt**：
```
查询海康威视(002415.SZ)，完整输出一份包含完整证据链的分析报告。

要求：
1. 获取该公司最近3年的主要财务指标
2. 计算"净现比" = 经营现金流 / 净利润
3. 计算"应收账款占比" = 应收账款 / 总营收
4. 基于这两个指标分析：该公司是否存在"赊销冲业绩"或"回款困难"？

关键要求：
- 每个财务指标都要明确标注 [DATE] 和 [SOURCE]
- 每个计算都要明确给出 [META] 定义（公式）
- 每个结论都要说明"基于的是哪个[DATA]数据"
- 如果无法获取某个数据，要说明 [WARNING] 和 [FALLBACK]方案

最终输出格式：
【数据来源与时间】
[DATE]: ...
[SOURCE]: ...

【计算逻辑】
[META]: 净现比 = ...（给出完整公式）
[META]: 应收账款占比 = ...（给出完整公式）

【具体数据】
[DATA]: 2023年净现比 = ...
[DATA]: 2024年净现比 = ...
[DATA]: 应收账款占比 = ...

【分析结论】
基于上述[DATA]数据，本公司...（每个结论都引用[DATA]）

【风险提示】
根据[META]和[DATA]，存在以下风险...
```

**预期验证点**：
- ✅ [DATE]、[SOURCE]、[META]、[DATA] 标签完整出现
- ✅ 没有凭空编造的数据（所有数据都可追溯）
- ✅ 如果数据缺失，有明确的 [WARNING] 和替代方案
- ✅ 结论段落中多次引用 [DATA] 或 [META]
- ✅ 用户可以按照 [META] 自己重新计算验证
