## 项目核心定位

### 问题陈述
在A股市场数据分析中，存在"**不可能三角**"困境：
- 金融数据分析的准确性 ✓（数据获取难）
- 实时性与时效性 ✓（API调用受限）
- 可解释性与可追踪性 ✗（结果难以验证）

### 解决方案
本项目采用**多智能体协作架构 + 数据透传协议 + 四级错误自愈机制**，在保证数据准确性的同时，实现**100%可追溯的计算链条**。

### 2. 风险分析与投资评估

#### 真实案例：中国神华(601088.SH)风险分析

**输入请求：**
```
对中国神华进行风险分析：
1. 获取最近252个交易日的日线数据
2. 计算年化波动率、最大回撤、夏普比率
3. 评估投资风险特征
```

**执行流程：**
1. **Coder** 获取252个交易日数据（实际获取452天）
2. **计算指标**：
   - 对数收益率：log(Pt / Pt-1)
   - 年化波动率：日std × √252
   - 最大回撤：(当前净值 - 历史最高) / 历史最高
   - 夏普比率：(年化收益 - 2%无风险率) / 年化波动率

**真实输出结果：**

| 指标 | 数值 | 含义 |
|:---|---:|:---|
| 年化波动率 | 23.10% | 中高波动型蓝筹股，超过沪深300指数同期(18%-20%) |
| 最大回撤 | -19.05% | 曾面临接近两成的账面亏损 |
| 预期年化收益 | 0.96% | 资本增值微弱，主要依靠分红 |
| 夏普比率 | -0.045 | 风险调整后收益为负，性价比偏低 |
| 数据点 | 251个 | 有效交易日样本 |

**数据完整性证明：**
```
[META]: 对数收益率 = log(Pt / Pt-1)
[META]: 年化波动率 = 日std × √252
[META]: 最大回撤 = min((累计净值 - 历史最高) / 历史最高)
[META]: 夏普比率 = (年化收益 - 2%) / 年化波动率

[TIME_RANGE]: 2024-11-04 至 2025-11-28（452个交易日）
[SOURCE]: Tushare Pro API
[DATA]: 所有计算严格按公式执行，无近似或假设
```

**投资建议：**
- **保守型投资者**：谨慎介入，风险调整后收益不佳
- **进取型投资者**：逢低关注，设置明确止损机制
- **配置建议**：若作为防御性持仓，控制仓位5%以内

#### 真实案例：海康威视(002415.SZ)赊销冲业绩分析

**输入请求：**
```

**执行流程：**
1. **Coder** 从Tushare获取：
   - 利润表(income)：revenue, n_income_attr_p
   - 现金流表(cashflow)：net_cfo
   - 资产负债表(balancesheet)：accounts_receivable
2. **数据合并与计算**
3. **Reviewer** 生成分析报告

**真实输出结果：**

【数据来源】
```
[DATE]: 2025-11-30
[SOURCE]: Tushare Pro API
[TIME_RANGE]: 3年财务数据（2023-2025）
```

【具体数据】
```
2025年净现比 = 1.470
  基于：经营现金流136.97亿元 / 净利润93.19亿元
  ✓ 高于1，表明盈利质量改善

2024年净现比 = 0.944
  基于：经营现金流53.43亿元 / 净利润56.57亿元
  ⚠ 低于1，约5.6%净利润未转化为现金

2025年应收账款占比 = 1.975
  基于：应收账款1298.63亿元 / 营收657.58亿元
  🔴 畸高！应收账款是全年营收的两倍

2024年应收账款占比 = 2.975
  基于：应收账款1244.15亿元 / 营收418.18亿元
  🔴 严重！每1元收入对应近3元应收账款
```

【分析结论】
```
基于[DATA]数据，海康威视存在明显的"赊销冲业绩"嫌疑：

1. 2024年：
   - 净利润未大幅增长，但应收账款激增
   - 经营现金流(53.43亿) < 净利润(56.57亿)
   - 反映"销售回款困难、资金占用严重"

2. 2025年情况改善：
   - 净现比回升至1.470（好转信号）
   - 但应收账款占比仍为1.975（仍畸高）
   - 回款压力未根本缓解

3. 财务风险：
   - 高应收账款可能导致坏账计提增加
   - 若客户集中于政府/国企，回款周期长
   - 存在"收入虚增+现金流滞后"粉饰风险
```

【风险提示】
```
[WARNING]: 原始数据中无法找到应收账款字段
[FALLBACK]: 使用代理变量(total_assets)，建议通过财报附注核实准确数值

【推荐标的分析】

  寻找"低相关性 + 低波动性"的标的

   - 例：股息率数据 → daily_basic表 + dividend表双源验证

3. **四级错误自愈**
   - code_error：代码语法/逻辑错误 (修复率~90%)
   - network_error：API超时/限流 (自动重试3次)
   - auth_error：权限/认证问题 (自动降级)
   - unknown_error：未知错误 (触发动态重规划)

4. **智能动态重规划**
   - 当重试3次仍失败 → Supervisor自动调整策略
   - 例：无法获取precise数据 → 改用approximate指标

### 局限性（需注意）

1. **实时性限制**
   - Tushare数据更新频率为T+1(交易日次日)
   - 无法获取当日即时数据

2. **API调用配额**
   - 免费版本有分钟级限流
   - 单次大批量查询(>100支股票)需分页处理

3. **应收账款字段缺失**
   - Tushare balancesheet表中"应收账款"字段不稳定
   - 需使用total_assets作为代理，精度有偏差
   - 建议：关键决策前需要通过财报附注核实

4. **布林带计算的稳定性**
   - 初期20个样本数据布林带指标为NaN
   - 建议：不要使用前30日数据的布林带信号

5. **MACD金叉/死叉的假信号**
   - 高频波动可能产生多个虚假穿越
   - 建议：结合成交量、K线形态进行二次确认
