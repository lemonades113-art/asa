# 金融Agent系统架构对比

## 1. 您的项目架构（LangGraph Supervisor模式）

```
用户查询
    ↓
┌─────────────────────────────────────────────────────────────┐
│                     Supervisor Node                         │
│  - 任务规划与拆解                                             │
│  - 维护任务队列: ["步骤1", "步骤2", "步骤3"]                   │
│  - 决策路由（3重验证机制）                                     │
└─────────────────────────────────────────────────────────────┘
    ↓ 条件路由                    ↓ 错误时
┌──────────────┐          ┌──────────────────┐
│  Coder Node  │          │ Error Handler    │
│  ----------  │          │ ----------------│
│  工具调用：   │          │  4层故障归因：   │
│  ✓ Tushare   │          │  1.代码错误      │
│  ✓ RAG检索   │          │  2.规划错误      │
│  ✓ Python    │          │  3.数据真空      │
│    执行内核   │          │  4.系统错误      │
│              │          │                 │
│  特性：       │          │  策略：          │
│  • 变量持久化 │          │  • 重新生成代码  │
│  • 用户隔离   │          │  • 切换策略      │
│  • 超时熔断   │          │  • 友好说明      │
└──────────────┘          └──────────────────┘
    ↓                            ↓
┌──────────────────────────────────────┐
│         Auditor Node                 │
│  三重验证：                           │
│  ✓ [DATA:] 标记存在                  │
│  ✓ 数据非空                          │
│  ✓ 结果完整（非部分）                │
│                                      │
│  → 通过：弹出任务队列，继续下一步     │
│  → 失败：保留任务，触发重试（最多3次）│
└──────────────────────────────────────┘
    ↓ 任务队列空
┌──────────────────┐
│ Profile Update   │
│  用户画像更新     │
└──────────────────┘
    ↓
最终结果返回用户
```

**核心创新点：**
- 状态机辅助约束（防止逻辑早停）
- 智能错误归因（避免盲目重试）
- 有状态执行内核（减少冗余查询）
- RAG混合检索（API文档查询）

## 2. PrimoAgent架构（LangGraph线性Pipeline）

```
用户查询（股票代码 + 日期）
    ↓
┌──────────────────────┐
│ Data Collection      │  ← yFinance + Finnhub API
│  获取：               │
│  • 实时行情           │
│  • 公司基本面         │
│  • 财务数据           │
│  • 新闻列表           │
└──────────────────────┘
    ↓ 固定流程（无条件路由）
┌──────────────────────┐
│ Technical Analysis   │
│  计算6个技术指标：     │
│  • SMA（趋势）        │
│  • RSI（超买超卖）    │
│  • MACD（动量）       │
│  • 布林带（波动率）   │
│  • ADX（趋势强度）    │
│  • CCI（周期指标）    │
└──────────────────────┘
    ↓
┌──────────────────────┐
│ News Intelligence    │
│  NLP特征提取（7维）： │
│  • sentiment（情感）  │
│  • price_impact（价格影响）│
│  • trend_direction（趋势）│
│  • earnings_impact（财报）│
│  • investor_confidence   │
│  • risk_profile_change   │
│  • news_relevance        │
└──────────────────────┘
    ↓
┌──────────────────────┐
│ Portfolio Manager    │
│  生成交易决策：        │
│  • BUY/SELL/HOLD     │
│  • 置信度（0.0-1.0）  │
│  • 仓位建议（%）      │
└──────────────────────┘
    ↓
结果保存为CSV → 用于回测
```

**特点：**
- ✅ 严格线性流程，易于理解
- ✅ 有完整的回测系统
- ❌ 无条件路由，灵活性差
- ❌ 无重试机制
- ❌ 无RAG集成

## 6. 您的项目独特优势

### 🏆 核心创新1：状态机辅助约束
**问题：** 所有其他项目都容易出现"逻辑早停"
```
场景：多步骤任务
步骤1：查询股票基本信息 ✓
步骤2：查询财务数据（Tushare返回空） ✗
步骤3：分析财务指标（未执行）

其他项目：LLM误判"任务完成"→ 直接返回
您的项目：Auditor检测到数据空 → 保留队列 → 重试
```

### 🏆 核心创新2：智能错误归因
**问题：** 其他项目"盲目重试"或"直接失败"
```
场景：字段名错误
代码：df = pro.daily(ts_code='000001.SZ', filed='close')  # 拼写错误
                                              ^^^^
其他项目：重试3次 → 仍然失败 → 浪费3次LLM调用
您的项目：错误归因 → RAG查询正确字段 → 1次纠正成功
```

### 🏆 核心创新3：有状态执行内核
**问题：** PrimoAgent每步重新调用API
```
场景：关联分析
步骤1：df1 = 查询利润表  ← API调用1
步骤2：df2 = 查询资产表  ← API调用2
步骤3：ratio = df1['净利润'] / df2['总资产']  ← 需要df1和df2

其他项目：步骤3需要重新调用API1和API2 → 冗余
您的项目：df1和df2持久化在内存 → 直接计算
```

### 从Phidata学习：
