# 多智能体金融分析系统 - 完整方案

## 核心问题：AI在金融的三角困境

通用LLM在金融领域无法解决三个根本问题：
- ❌ **时效性**：无法获取实时数据（LLM无网络能力）
- ❌ **可验证性**：黑盒决策，用户无法验证推理过程
- ❌ **鲁棒性**：一个错误就停止，无法自动修复

---

## 解决方案：5个专业智能体的虚拟投研团队

### 架构核心

| 角色 | 职责 | 本质 |
|:---|:---|:---|
| **Supervisor** | 任务分解 + 路由 + 质检 | 大脑：用状态机替代LLM自由决策，避免虚假完成 |
| **Coder** | Python代码生成 + API调用 | 手脚：执行计算，强制输出[DATE][SOURCE][META][DATA] |
| **Reviewer** | 报告撰写 + 数据验证 | 眼睛：基于Coder的透传数据生成分析 |
| **ErrorHandler** | 错误诊断 + 自动修复 | 免疫系统：4级自愈，95%修复成功率 |
| **ProfileUpdater** | 用户画像 + 个性化 | 记忆：学习用户偏好，改进服务 |

## 核心创新点

### 1. 数据完全透传（解决可验证性）

**问题**：用户问"为什么推荐格力电器？"，但系统只能说"我分析后认为..."

**解决**：强制三层数据流

```python
# Layer 1: Meta（定义层）
[DATE]: 2025-11-28
[SOURCE]: Tushare Pro API
[META]: 波动率 = 日std × √252
[META]: 综合评分 = 70%×股息率 + 30%×波动率

# Layer 2: Data（结果层）
[DATA]: {"格力": {stock_dividend: 7.41, volatility: 0.1665, score: 8.64}}

# Layer 3: Report（解释层）
Reviewer基于[META]和[DATA]写报告：
"格力电器股息率7.41%、波动率0.1665%...综合评分8.64，排名第一"
```

**价值**：用户可以完全验证
- ✓ "我打开Tushare查2025-11-28数据" → 验证[DATA]
- ✓ "我用日std×√252计算" → 验证[META]
- ✓ "我用70%+30%权重" → 验证计算逻辑

---

### 2. 四级错误自愈（解决鲁棒性）

**问题**：API字段变更、网络限流、数据缺失 → 系统停止

**解决**：分级处理机制

```
Level 1: Coder自救
  ├─ 代码语法错误 → 修复代码再执行
  └─ 数据类型错误 → 转换类型后重试

Level 2: ErrorHandler诊断
  ├─ code_error → 建议字段映射修复
  ├─ network_error → 指数退避重试
  ├─ auth_error → Token刷新
  └─ unknown_error → 降级方案

Level 3: 自动修复
  └─ 尝试备选方案，例：
     ✗ 原: df['accounts_receivable']
     ✓ 修: 尝试['ar','receivable','note_receivable']

Level 4: 动态重规划
  └─ 3次重试失败 → Supervisor调整策略
```

**验证数据**：
- 309支股票处理 × 6步骤 = 1854个潜在故障点
- 自动修复：API限流4次、字段缺失2次、网络超时1次 = **7次故障自动修复**
- 成功率：**100%任务完成，95%无人工干预**

---

### 3. 状态机路由（解决时效性）

**问题**：单体Agent会"虚假完成"（说FINISH但未完成）、"无限循环"、"跳步"

**解决**：用可验证的规则替代LLM决策

```python
# 二维决策空间：last_sender + execution_status

if last_sender == "Coder" and execution_status == "success":
    if remaining_steps > 0:
        next = "Coder"      # 继续执行下一步
    else:
        next = "Reviewer"   # 所有步骤完成，派给报告
        
if last_sender == "Coder" and execution_status == "error":
    next = "ErrorHandler"   # 出错立即诊断
    
# 保证：
✓ 不会虚假完成（remaining_steps控制）
✓ 不会无限循环（每步pop一个step）
✓ 不会跳步（严格按顺序执行）
```

**验证数据**：
- 沪深300筛选案例：6步任务分解 → 6步完整执行
- 309支股票处理：没有任何虚假完成或无限循环
