## 核心能力方向

### 📊 系统主要擅长的四大领域

#### **1. 数据获取与清洗** ⭐⭐⭐⭐⭐

**能力描述**：
- 从Tushare API实时获取A股市场数据
- 自动处理数据类型转换和兼容性问题
- 支持多维度数据汇总和交叉引用

**支持的数据类型**：
- 日线数据（开高低收成交量）
- 估值指标（PE、PB、市值）
- 财务报表（收入、利润、资产）
- 分红数据（每股分红、分红率）
- 行情数据（涨跌幅、成交额）
- 指数成分股权重

**典型应用**：
```
✅ "查询平安银行最近3个月的日线数据"
✅ "列出沪深300中市盈率最低的20支股票"
✅ "获取工商银行过去5年的分红数据"
✅ "汇总金融行业上市公司的财务指标"
```

---

#### **2. 财务分析与估值** ⭐⭐⭐⭐

**能力描述**：
- 计算关键财务指标（PE、PB、ROE、股息率等）
- 进行历史对标和行业对比
- 评估股票的估值水平（高估/低估/合理）

**支持的分析类型**：
- 单股财务评估
- 行业均值对标
- 同业竞争对比
- 估值历史趋势
- 分红稳定性评估

**典型应用**：
```
✅ "分析贵州茅台的当前PE水平是否高估"
✅ "对比三支银行股的ROE和分红率"
✅ "评估某股票相对行业的估值位置"
✅ "计算过去5年的分红率变化趋势"
```

---

#### **3. 投资决策支持** ⭐⭐⭐⭐

**能力描述**：
- 基于多维指标进行股票评分和排名
- 为不同风险偏好的投资者提供选股建议
- 设计投资组合配置方案

**支持的决策类型**：
- 选股筛选（按指标条件）
- 投资评分（多指标加权）
- 组合配置（资产配置建议）
- 风险评估（波动率、最大回撤）
- 个性化推荐（基于用户画像）

**典型应用**：
```
✅ "为保守型投资者推荐高股息股票"
✅ "设计一个兼顾成长和分红的股票组合"
✅ "评估某投资策略的历史表现"
✅ "基于不同行业的估值水平给出配置建议"
```

---

#### **4. 报告生成与解释** ⭐⭐⭐⭐

**能力描述**：
- 自动撰写专业的金融分析报告
- 清晰解释复杂的数据和模型
- 提供投资结论和风险提示

**支持的报告类型**：
- 个股分析报告
- 行业评估报告
- 投资组合设计报告
- 策略回测总结
- 择股方法论说明

**典型应用**：
```
✅ "生成某只股票的完整分析报告"
✅ "撰写行业景气度评估"
✅ "输出投资组合的配置方案和理由"
✅ "解释某择股策略为什么有效"
```

## Prompt难度分级体系

### 🎓 四个难度级别的分类标准

| 级别 | 名称 | 特征 | 成功率 | 执行模式 |
|------|------|------|--------|---------|
| **L1** | 初级查询 | 单步操作，无复杂分析 | 95%+ | Coder单步 |
| **L2** | 中级分析 | 多步数据处理，初步分析 | 85%-90% | Coder→Reviewer |
| **L3** | 高级综合 | 多维分析，完整报告 | 70%-80% | 完整流程 |
| **L4** | 深度建模 | 复杂模型，多轮交互 | 60%-75% | 可能多轮纠正 |

---

## 各难度级别详解

### 🌟 L1级：初级查询型 (95%+ 成功率)

#### **级别定义**
- **复杂度**：⭐ 最低
- **特点**：单一API调用，无需复杂分析
- **智能体参与**：Coder (仅需执行代码)
- **预计耗时**：5-10秒

#### **执行流程**

```
用户问题 → Supervisor(识别) → Coder(执行API) → 输出数据 → FINISH
```

#### **适用场景**

| 场景 | 示例Prompt | 输出内容 |
|------|----------|--------|
| **单股查询** | "查询平安银行(000001.SZ)最新的PE、PB、股息率" | 表格（3个指标） |
| **简单列表** | "列出沪深300中股息率最高的10支股票" | 列表（10行数据） |
| **行业筛选** | "查询金融行业所有股票的最新收盘价" | 数据集（行业内全部股） |
| **单指标计算** | "计算贵州茅台过去30天的累计涨幅" | 单个数值+图表 |

#### **关键特征**

✅ **能做**：
- 调用单个API接口
- 简单的数据列表展示
- 单个指标的查询和计算

❌ **不需要做**：
- 跨多个API的复杂组合
- 数据之间的对比分析
- 投资建议生成

#### **成功的Prompt特点**

```
好的L1 Prompt：
1. 明确指定股票代码（000001.SZ格式）
2. 清晰列出要查询的指标（PE、PB、股息率）
3. 不要求任何分析或建议
4. 数据范围清晰（最新、过去30天等）

示例：
"请查询平安银行(000001.SZ)、招商银行(600036.SH)、建设银行(601939.SH)
的最新PE、PB、股息率、市值四个指标。"
```

#### **失败的Prompt特点**

```
不好的L1 Prompt：
1. 模糊的概念（"热门股票"、"好股票"）
2. 要求分析和建议（"哪个更好？"）
3. 跨越多个数据域（"获取股价+财报+分红"）

示例：
"帮我找一个好的银行股，成长性要好，分红也要高。"
→ 这太模糊，会升级到L2或L3
```

---

### ⭐⭐ L2级：中级分析型 (85%-90% 成功率)

#### **级别定义**
- **复杂度**：⭐⭐ 中等
- **特点**：多步骤数据处理，简单对比分析
- **智能体参与**：Supervisor + Coder + Reviewer
- **预计耗时**：10-20秒

#### **执行流程**

```
用户问题 
  → Supervisor(任务分解：3-5步)
  → Coder(执行多个API调用，计算指标)
  → Reviewer(生成初步分析报告)
  → FINISH
```

#### **适用场景**

| 场景 | 特征 | 示例 |
|------|------|------|
| **股票对标** | 2-5支股票间的对比分析 | "对比三支银行股的估值和分红" |
| **行业均值** | 某行业的平均水平评估 | "计算地产行业的平均PE和PB" |
| **技术分析** | 简单的价格和波动率分析 | "分析某股票最近6个月的走势和波动率" |
| **回测验证** | 简单策略的历史验证 | "验证买低估股票的历史收益" |

#### **关键特征**

✅ **能做**：
- 批量获取多支股票数据
- 计算多个维度的指标
- 进行简单的对标对比
- 生成结构化的分析表格

❌ **仍不需要做**：
- 复杂的财务建模（如DCF估值）
- 跨年份的历史数据汇总
- 详细的风险分析

#### **成功的Prompt特点**

```
好的L2 Prompt：
1. 明确指定分析对象（2-5支股票或某行业）
2. 指定对比维度（估值、成长、分红）
3. 要求简单的结论（"哪个更划算"）
4. 尽量包含业务背景（"我是保守型投资者"）

示例：
"请对比平安银行(000001.SZ)、招商银行(600036.SH)、建设银行(601939.SH)
这三支股票的估值水平(PE、PB)、分红率和最近3个月的涨幅。
告诉我哪个更值得买。"
```

#### **失败的常见原因及应对**

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| "某支股票数据为空" | 股票无该数据或API失败 | 系统会自救换API，或跳过该股 |
| "超时卡住" | API限流或网络延迟 | 系统会自动重试或降级 |
| "结论相反" | Reviewer理解有偏差 | 检查你的表述是否清晰 |

---

### ⭐⭐⭐ L3级：高级综合型 (70%-80% 成功率)

#### **级别定义**
- **复杂度**：⭐⭐⭐ 较高
- **特点**：完整的两轮分析流程，可能需要自救
- **智能体参与**：所有5个智能体可能都参与
- **预计耗时**：20-35秒

#### **执行流程**

```
用户问题
  → Supervisor(任务分解：5-10步)
  → Coder(第1轮：获取基础数据)
    → 如果数据不足 → ErrorHandler(尝试自救)
      → Coder(第2轮：使用自救方案重新获取)
  → Reviewer(撰写详细分析报告)
  → ProfileUpdater(学习用户画像)
  → FINISH
```

#### **适用场景**

| 场景 | 特征 | 示例 |
|------|------|------|
| **行业评估** | 评估整个行业的景气度 | "评估电力行业(发电、电网、新能源)的当前景气度" |
| **选股策略** | 按条件筛选最优股票 | "从沪深300中筛选高股息且低波动的5支股票" |
| **投资主题** | 分析某个投资主题的机会 | "分析新能源汽车产业链的投资机会" |
| **组合设计** | 为特定人群设计投资组合 | "为风险厌恶型投资者设计股债配置方案" |

#### **关键特征**

✅ **能做**：
- 5-20支股票的大规模数据采集
- 多维度的评分和排名
- 生成专业的分析报告
- 自动学习用户偏好

⚠️ **可能会**：
- 触发API限流（需要自救降级）
- 需要1-2次自纠正
- 对网络环境有一定依赖

❌ **仍不能做**：
- 超过20支股票的大规模筛选（会超时）
- 跨越多年财报的深度历史分析
- 实时预测或趋势判断

#### **成功的Prompt特点**

```
好的L3 Prompt：
1. 明确的分析目标（行业/主题/策略）
2. 明确的评选条件（股息率>4%, 波动率<0.15等）
3. 预期输出形式（评分表、Top 5推荐等）
4. 业务背景说明（我的风险偏好是...）

示例：
"我是一个保守型投资者，想建立一个'高股息股票筛选器'来寻找稳健的分红股。
请从沪深300中筛选股息率最高的5支股票，并基于股息率(60%)和波动率(40%)
为它们打分，给出Top 2推荐。"
```

#### **可能的失败场景及自救**

| 失败原因 | 系统自救 | 最终结果 |
|---------|--------|--------|
| 沪深300数据获取返回6000支 | 添加limit=300限制 | 继续执行 |
| 某支股票无分红数据 | 跳过该股或换API | 基于可用数据分析 |
| API限流（获取500支股票） | 降级为先获50支，计算其他 | 基于样本输出结果 |
| Reviewer第1次生成报告失败 | 自动重试最多3次 | 成功或输出备用方案 |

### ⭐⭐⭐⭐ L4级：深度建模型 (60%-75% 成功率)

#### **级别定义**
- **复杂度**：⭐⭐⭐⭐ 很高
- **特点**：多轮复杂分析，涉及财务建模或历史数据分析
- **智能体参与**：所有智能体，可能多次交互
- **预计耗时**：30-50秒

#### **执行流程**

```
用户问题
  → Supervisor(复杂任务分解：8-15步)
  → Coder(第1轮：获取基础数据)
    → 如果数据不足 → ErrorHandler
      → Coder(第2-3轮：多次自救)
  → Reviewer(尝试生成报告)
    → 如果失败 → 重试+降级
  → ProfileUpdater(深度学习用户特征)
  → FINISH或回到Supervisor重新规划
```

#### **适用场景**

| 场景 | 特征 | 示例 | 风险 |
|------|------|------|------|
| **财务评估** | 3年财报分析 + 估值建模 | "对平安银行做完整价值评估" | 数据滞后 |
| **股债配置** | 100万资金的配置方案 | "设计股债组合并模拟压力测试" | 债券数据不全 |
| **行业轮动** | 多行业的对比和配置建议 | "分析当前应该布局哪个行业" | 行业分类困难 |
| **风险回测** | 某策略的历史表现验证 | "验证'高分红+低波动'策略的回测结果" | 数据覆盖不完整 |

#### **关键特征**

✅ **能做**：
- 3年以上的财务数据分析
- 多个行业的对标对比
- 复杂的评分模型设计
- 历史数据的回测计算

⚠️ **可能遇到的问题**：
- 财报数据只有2023年完整年报（2024+需用TTM）
- API可能限流（需多次自救）
- Reviewer报告生成可能失败（需3次重试机制）
- 概念板块分类缺失（如"AI芯片"需手动指定股票）

❌ **确定做不了**：
- 未来趋势预测
- 实时行情预警
- 期货/衍生品分析
- 超过1年的完整年报数据

#### **成功的Prompt特点**

```
好的L4 Prompt：
1. 明确的复杂目标（财务评估、模型设计等）
2. 清晰的数据范围和时间周期
3. 预期模型的具体细节（权重、评分方法等）
4. 充分的业务背景

示例：
"请对平安银行(000001.SZ)做一个完整的价值评估：
1. 获取最近可用的财务报表(收入、利润、ROE等)
2. 计算其PE、PB、ROE的历史趋势
3. 与同业对比(招商、工商、建设)
4. 计算合理估值区间
5. 给出投资建议
"
```

#### **降级处理说明**

```
当L4任务遇到问题时，系统会自动降级处理：

例：财报数据只有2023年
  → 自动使用TTM数据（Trailing Twelve Months）
  → 获取最新季报补充
  → 基于可用数据完成分析

例：某行业数据不全
  → 扩大样本范围
  → 使用相似行业对标
  → 基于可用数据输出结论
```

### ⭐⭐⭐⭐⭐ L5级：特殊高阶型 (40%-50% 成功率) 🔴 不推荐

#### **级别定义**
- **复杂度**：⭐⭐⭐⭐⭐ 极高
- **特点**：多轮用户交互，系统动态学习调整
- **智能体参与**：所有智能体反复交互
- **预计耗时**：60秒+

#### **不推荐的原因**

| 问题 | 表现 | 影响 |
|------|------|------|
| 多轮反馈不稳定 | 第2轮、第3轮的表现下降 | 用户体验差 |
| 概念理解困难 | 如"AI芯片"无自动分类 | 需要用户逐个指定 |
| 动态学习滞后 | ProfileUpdater更新但不主动重规划 | 需用户重新表述 |
| 消息膨胀 | 多轮交互导致消息历史爆炸 | 容易超过Token限制 |

#### **例子：为什么L5不推荐**

```
第1轮：用户问 "请分析贵州茅台"
→ 系统输出报告，更新画像为"白酒+成长"

第2轮：用户说 "其实我对消费股更感兴趣"
→ 系统理解了，但不会主动重新筛选
→ 需要用户明确说："请筛选消费股而不是白酒"

第3轮：用户说 "食品和日用品最好，不要酒类"
→ 这时多轮交互的消息已经膨胀
→ Reviewer可能开始出错或超时

结论：L5级别的多轮优化通常以失败告终
```

## 能力边界与限制

### 🚫 系统明确不支持的功能

#### **数据源层限制**

| 不支持 | 原因 | 可能的替代方案 |
|--------|------|-----------|
| 期货/期权数据 | Tushare无此接口 | 使用单独的期货API |
| 加密货币数据 | Tushare无此接口 | 使用CoinGecko等专门API |
| 海外市场数据 | Tushare仅覆盖A股 | 使用Yahoo Finance等 |
| 实时tick级数据 | Tushare无此接口 | 订阅专业行情源 |
| 机构持仓详细数据 | API限制 | 手动获取披露信息 |

#### **分析能力层限制**

| 不支持 | 原因 | 备注 |
|--------|------|------|
| 趋势预测 | 系统基于历史数据，无预测能力 | "未来会涨"类问题做不了 |
| 实时行动建议 | 需要实时反馈和风险评估 | "现在买还是卖"需人工决策 |
| 跨资产配置 | 仅有股票数据，债券数据不全 | 股债配置会降级 |
| 自动择时 | 无技术面预测能力 | 波段操作需人工判断 |
| 风险对冲 | 无衍生品分析 | 对冲策略无法设计 |

#### **场景适用性限制**

| 不适合 | 原因 | 建议 |
|--------|------|------|
| 日内交易分析 | 系统基于日线数据 | 需要专业行情软件 |
| 套利策略设计 | 无多市场数据关联 | 需要人工分析 |
| 量化模型构建 | 仅支持简单评分模型 | 需要专业量化平台 |
| 实时风险监控 | 无实时数据更新 | 需要专业风控系统 |
| 个人账户管理 | 无持仓跟踪功能 | 需要专业投资工具 |

### ⚠️ 常见的"能做但有风险"的应用

#### **场景1：财务数据时效性问题**

```
问题：我想分析某公司的2024年财务情况

系统现状：
- 2023年完整年报 ✅ 可用
- 2024年1-3季度季报 ✅ 可用
- 2024年完整年报 ❌ 不可用(未披露)

自救方案：
1. 使用TTM(Trailing Twelve Months)数据
2. 基于最新季报外推
3. 明确标注"基于TTM/季报估算"

结论：能做，但要标注数据限制
```

#### **场景2：概念板块识别问题**

```
问题：帮我分析"AI芯片"相关公司的机会

系统现状：
```

## 最后的建议

### 🎓 如何充分利用这个系统

1. **明确你的问题难度**
   - 问题是L1还是L3？
   - 按建议难度级别设计Prompt

2. **提供充分的上下文**
   - 明确的数据范围
   - 明确的分析维度
   - 你的业务背景

3. **相信系统的自救机制**
   - 如果第一次失败，系统会重试
   - 如果数据不足，系统会降级
   - 不需要你手动干预

4. **合理规划使用频率**
   - L1：随时可用（快速）
   - L2-L3：日常使用（中等复杂）
   - L4：重要决策（谨慎使用）
   - L5：避免使用

5. **记住系统的核心优势**
   - ✅ 数据获取和清洗能力强
   - ✅ 分析报告生成能力强
   - ✅ 自动学习用户偏好
   - ❌ 多轮交互不稳定
   - ❌ 无预测能力
