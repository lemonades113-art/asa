## 技术分析

### 字符串定界符

Python 的多行字符串使用三引号定界：

```python
CODER_SYSTEM_PROMPT = """
# 内容很长...
第 1317 行: print(f"[DATA]: {json.dumps(result, ensure_ascii=False)}")
   ```
"""  # ← 这个 """ 必须存在！
```

### 为什么之前会出错

当修改在第 1252-1318 行之间插入大量新代码（净利润与毛利率分析部分）时，**遗漏了最后的三引号关闭符**。导致：

1. CODER_SYSTEM_PROMPT 字符串一直延伸到 REVIEWER_SYSTEM_PROMPT 开始
2. REVIEWER_SYSTEM_PROMPT 的内容（包含 Emoji）被 Python 当成代码
3. Emoji 字符不被认为是代码中的有效字符，抛出 SyntaxError
