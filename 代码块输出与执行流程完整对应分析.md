## 📋 总体架构回顾

```
User Input
    ↓
[supervisor_node] → 决策下一步
    ↓
分支1: Coder   分支2: Reviewer   分支3: FINISH
    ↓ (工具调用)
[execute_tools] → 执行代码，生成ToolMessage
    ↓
[error_handler_node] → 判断错误，决定重试或回到Supervisor

    ↓
[profile_updater_node] → 更新用户画像
    ↓
回到 [supervisor_node] → 决策是否FINISH
    ↓
END
```

```python
def classify_error_simple(error_str: str) -> str:
    """快速错误分类"""
    keywords = {
        "code_error": ["SyntaxError", "NameError", "TypeError", "IndentationError", "AssertionError"],
        "network_error": ["timeout", "Connection refused", "HTTPError", "429", "503"],
        "auth_error": ["401", "403", "Unauthorized", "token", "permission"],
    }
    
    for error_type, keywords_list in keywords.items():
        if any(kw in error_str for kw in keywords_list):
            return error_type
    
    return "unknown_error"
```

### 代码块2：分级处理与重试
```python
# multi_agent.py L1491-1533
retry_count = state.get("retry_count", 0)
network_retry_count = state.get("network_retry_count", 0)

if error_type == "code_error" and retry_count < 3:
    # 🔄 代码错误：最多重试3次
    print(f"[ErrorHandler] [代码错误] 第{retry_count + 1}次重试...")
    return {
        "next": "Coder",
        "retry_count": retry_count + 1,
        "execution_status": "retrying"
    }

elif error_type == "network_error" and network_retry_count < 3:
    # 🔄 网络错误：指数退避重试
    wait_time = 2 ** network_retry_count  # 1, 2, 4, 8秒
    print(f"[ErrorHandler] [网络错误] 等待{wait_time}秒后重试...")
    import time
    time.sleep(wait_time)
    return {
        "next": "Coder",
        "network_retry_count": network_retry_count + 1,
        "execution_status": "retrying"
    }

elif error_type == "auth_error":
    # ❌ 认证错误：直接失败，不重试
    print(f"[ErrorHandler] [认证错误] API密钥无效，终止任务")
    return {
        "next": "FINISH",  # 直接END，不重试
        "execution_status": "failed",
        "error_type": "auth_error"
    }

elif retry_count >= 3 or network_retry_count >= 3:
    # ⚠️ 重试耗尽：回到Supervisor重规划
    print(f"[ErrorHandler] ⚠️ 重试耗尽(retry_count={retry_count}, network_retry={network_retry_count})")
    return {
        "next": "Supervisor",  # 由Supervisor负责重规划
        "execution_status": "retry_exhausted"
    }
```

**决策矩阵**：

| 错误类型 | 重试次数 | 行动 | 下一步 |
|---------|--------|------|-------|
| code_error | < 3 | 重试 Coder | Coder |
| code_error | ≥ 3 | 回到Supervisor | Supervisor |
| network_error | < 3 | 指数等待后重试 | Coder |
| network_error | ≥ 3 | 回到Supervisor | Supervisor |
| auth_error | - | 直接失败 | FINISH |
| unknown_error | < 3 | 重试 | Coder |
| unknown_error | ≥ 3 | 回到Supervisor | Supervisor |

**执行流程中的位置**：
```
Tools执行完毕，返回ToolMessage
    ↓
ErrorHandler检查内容
    ├─ ✅ 没有"Error"或"Traceback" → 成功，送Supervisor
    └─ ❌ 包含错误信息 → 进行分类
    ↓
【分类】根据关键字判断错误类型
    ├─ SyntaxError → code_error
    ├─ timeout → network_error
    └─ 401/403 → auth_error
    ↓
【决策】根据类型和重试次数
    ├─ code_error × 3 → 重新派给Coder
    ├─ network_error × 3 + 指数等待 → 重新派给Coder
    ├─ auth_error → 直接FINISH
    └─ 重试超过3次 → 派给Supervisor重规划
```
