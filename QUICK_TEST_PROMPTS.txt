## 💡 关键观察点

运行时，特别关注：

1. **Supervisor 的任务分解是否合理**
   - 是否正确识别多步骤任务
   - 是否自动调用 Coder 和 Reviewer

2. **Coder 的代码执行能力**
   - Python 代码是否正确
   - 是否能正确使用 Pandas、Numpy
   - 是否能正确调用 Tushare API

3. **Reviewer 的专业度**
   - 分析是否有深度
   - 是否基于数据说话
   - 格式和逻辑是否清晰

4. **错误处理能力**
   - API 限流时是否自动重试
   - 异常数据是否被正确处理
   - 是否有合理的降级方案

5. **对话连贯性（L5）**
   - 是否记住了用户反馈
   - 是否学到了用户偏好
   - 后续建议是否被相应调整

## 🎯 快速诊断

| 现象 | 问题 | 解决 |
|------|------|------|
| L1 失败 | API 连接问题 | 检查 conf.py，运行 verify_aliyun_config.py |
| L2 失败 | Supervisor 路由问题 | 检查日志，看是否进入 Coder/Reviewer |
| L3 失败 | 复杂任务分解失败 | Supervisor 可能没有正确识别多步骤 |
| L4 失败 | 没有自动重试 | ErrorHandler 可能未正常工作 |
| L5 失败 | 多轮对话记忆问题 | ProfileUpdater 可能未保存用户偏好 |
